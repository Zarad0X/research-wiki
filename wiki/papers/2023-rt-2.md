# RT-2

## Summary

RT-2 是这条线里的 landmark paper。它把 web-scale 预训练的 vision-language model 直接改造成可以输出机器人动作的 vision-language-action model，并且展示了这种做法不仅能做控制，还能把一部分语义知识和泛化能力迁移到机器人上。

## Problem

- 机器人数据很贵，不可能覆盖所有物体、环境和任务
- 单纯靠机器人 demonstration 学到的 policy 很难获得 web-scale 语义泛化
- 需要一种方法把 VLM 的知识直接迁移到端到端控制里

## Main Idea

- 从已经有 web-scale 视觉语言知识的 VLM 出发
- 在机器人轨迹数据和互联网视觉语言任务上联合微调
- 把机器人动作表示成文本 token，让模型继续沿用语言建模的输出接口

## Method

- 论文把这类模型命名为 `vision-language-action model (VLA)`
- 使用离散化动作，并把动作序列编码成类似字符串的 token 序列
- 以 PaLI-X 和 PaLM-E 为 backbone，得到 RT-2 的不同实例

## Experimental Setup

- 建立在 RT-1 的机器人 demonstration 数据之上
- Google DeepMind 博客写到，RT-1 数据来自 13 台机器人、17 个月的 office kitchen 采集
- 论文与项目页报告了 6000+ 次机器人试验，以及 Language Table 的 sim 和 real-world 评测

## Results

- 在 emergent skill evaluation 上，相比基线有超过 3x 的提升
- 对未见场景的泛化从 32% 提升到 62%
- 在 Language Table simulation 上达到 90% success rate，高于 BC-Z、RT-1、LAVA 等基线

## Strengths

- 非常清晰地定义了 VLA 这个范式
- 方法改动相对直接，核心直觉很强：把 action 也纳入 token space
- 证明 web pretraining 不只是“看懂”，还能帮助 robot control 的泛化和语义推理

## Limitations

- 整体体系偏 closed，复现和二次研究门槛高
- 动作离散 token 化虽然方便统一接口，但可能限制高频、精细、连续控制
- 主要证明的是 manipulation/generalization 潜力，不等于已经解决现实机器人部署问题

## Questions

- RT-2 的效果有多少来自 backbone 本身，有多少来自数据和训练 recipe
- 离散 action token 在更 dexterous、更长时域任务中是否会成为瓶颈
- 这种“语言 token 化动作”的设计，和后来的 flow matching 路线相比谁更可扩展

## Initial Take

初读视角下，RT-2 更像“定义时代的论文”而不是“最适合你直接复现实验的论文”。它最重要的贡献，是把 VLA 这件事讲通了，并证明这不是单纯的 marketing phrase，而是一个可工作的建模范式。
