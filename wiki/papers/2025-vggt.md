# VGGT

## Summary

`VGGT` 最值得重视的地方，不只是它在多个 3D benchmark 上刷了更好的数字，而是它把一个很老的 computer vision 命题重新讲成了 foundation model 叙事：scene geometry 是否也可以像语言、图像那样，被一个足够大的通用 transformer 直接前馈预测出来，而不再严重依赖 test-time geometry optimization。作者给出的答案相当激进：用一个约 `1.2B` 参数、几乎没有显式 3D inductive bias 的大模型，直接从 `1`、`few` 到 `hundreds` 张图像中，一次性预测场景的关键几何属性，包括 camera intrinsics / extrinsics、depth maps、point maps 和 tracking features。

如果这个叙事成立，`VGGT` 的意义就不只是“一个更快更强的 SfM/MVS 模型”，而更像是 `3D geometry foundation model` 的雏形。它说明多视图几何也许可以被统一进一个共享 backbone，让 camera estimation、depth、point cloud reconstruction、point tracking 这些传统上拆开的任务一起学、一起涨，甚至把训练好的特征拿去增强下游非刚体 tracking 和 feed-forward novel view synthesis。

## Story / Setting

- 传统 3D reconstruction 长期依赖经典视觉几何 pipeline：matching、triangulation、bundle adjustment、multi-view stereo。
- 深度学习虽然逐步替换了其中的一些组件，但大多数系统仍然 heavily depend on post-processing optimization。
- `DUSt3R` / `MASt3R` 这代方法已经往 feed-forward 方向走了一步，但它们通常仍围绕 pairwise setup 构造，再依赖对齐或其他后处理把多视图拼起来。
- `VGGT` 想问的是：如果不把这些任务拆散，而是直接把“一个场景的几何状态”当成大型多任务前馈模型的输出，会不会更简单、更快，还更强。

## Why This Exists

- 想进一步去掉 3D vision 里昂贵而复杂的 test-time optimization。
- 想让 camera pose、depth、point map、tracking feature 共用一个 backbone，而不是每个任务一个专门模型。
- 想证明 geometry 也可以走大模型路线：少 inductive bias，更多数据，更统一的任务建模。
- 想把 3D reconstruction 从一个繁琐 pipeline 变成一个可复用 backbone。

## Related Work

- 经典 `SfM / BA / MVS` 方法代表的是 optimization-first 世界。
- `VGGSfM` 这种可微 pipeline 是 machine learning 和 visual geometry 更深度融合的一步，但视觉几何后处理仍然很重。
- `DUSt3R` / `MASt3R` 代表 feed-forward multi-view geometry 的前一波高潮，但它们仍偏 pairwise and alignment-heavy。
- `Depth Anything`、`MoGe`、`LRM` 这类更大的 3D neural network 往往只盯单一任务，比如 monocular depth 或 novel view synthesis。
- `VGGT` 的位置很明确：不是单任务 geometry model，而是尝试做一个联合预测多个关键 3D quantity 的通用 backbone。

## First Principles

- 一个场景的 camera、depth、point map、tracks 不是彼此独立的任务，而是同一个几何状态的不同投影。
- 如果模型内部真的学到了稳定的 scene geometry representation，那么这些相关输出最好一起预测，而不是分别学。
- 传统 3D 视觉一直默认 geometry reasoning 必须在 test-time optimization 里发生；`VGGT` 则在赌，这种 reasoning 可以大部分在 pretraining / training 阶段被编译进前馈网络。
- 这也意味着 geometry 可能像语言和图像一样，逐渐从 “pipeline skill” 变成 “backbone capability”。

## Problem

- 如何从任意数量、多视角的一组图像里，稳定估计场景的核心 3D attributes。
- 如何避免只会处理成对图像，或者只在需要昂贵后处理时才有效。
- 如何把 camera pose、depth、point map、tracking 统一进同一个模型里，而不是性能互相拖累。
- 如何让模型既在 in-domain benchmark 上强，也能在更加野生的图像条件下保持泛化。

## Main Idea

- 用一个大规模 feed-forward transformer 直接 ingest 多张场景图像。
- 用共享 backbone 联合预测：
  - camera extrinsics / intrinsics
  - depth maps
  - point maps
  - dense tracking features
- 架构上只保留很弱的几何归纳偏置，主要依赖多任务学习和大规模 3D annotated data 来学出 geometry reasoning。
- 在 inference 时，可以直接使用前馈输出；如果愿意，也可把预测结果再喂给 BA 做更高精度 refinement。

## Core Architecture

- `DINO` patchify 输入图像，生成 image tokens。
- 每帧额外添加 `camera token` 和 `register tokens`，帮助相机预测与参考系建模。
- 主干采用 `Alternating Attention`：交替执行 `frame-wise self-attention` 与 `global self-attention`。
- `camera head` 负责预测 intrinsics / extrinsics。
- `DPT head` 负责预测 depth、point maps 和 dense tracking features。
- tracking 使用类似 `CoTracker2` 的 head，在特征图上对 query point 做跨帧关联。
- 所有预测默认都放在第一帧的 camera coordinate frame 中表达。

## Method

- 使用多任务 loss 联合监督 camera、depth、point map、tracking。
- camera 采用 Huber loss。
- depth / point map 使用 uncertainty-aware loss，depth 额外加了 gradient-based term。
- tracking 分支与 backbone 端到端联合训练，并预测 visibility。
- 训练时先做 coordinate normalization，把所有 3D quantity 统一到第一帧坐标系，并做尺度规范化。
- 论文还观察到：虽然训练时显式监督 point maps 有帮助，但推理时用 `predicted depth + predicted cameras` 反投影得到的点云更准。

## Experimental Setup

- 模型规模约 `1.2B` 参数。
- 训练使用 `64 x A100`，持续约 `9` 天。
- 训练数据是一个很大的 3D-annotated mixture，包括 `Co3Dv2`、`BlendMVS`、`DL3DV`、`MegaDepth`、`Kubric`、`WildRGB`、`ScanNet`、`HyperSim`、`Mapillary`、`Habitat`、`Replica`、`MVS-Synth`、`PointOdyssey`、`Virtual KITTI`、`Aria Synthetic Environments`、`Aria Digital Twin`，以及一个类似 `Objaverse` 的 synthetic assets dataset。
- 数据覆盖 real / synthetic、indoor / outdoor、多种 3D 标注来源，这正是它 foundation-style 叙事的重要支撑。

## Results

- 在 `RealEstate10K` 和 `CO3Dv2` 上，10-frame camera pose estimation 的 feed-forward `AUC@30` 达到 `85.3 / 88.2`。
- 若与 BA 结合，`AUC@30` 可进一步提升到 `93.5 / 91.8`。
- 在 `DTU` 上，即使不知道 GT cameras，dense MVS overall score 也做到 `0.382`，显著优于 `DUSt3R` 的 `1.741`。
- 在 `ETH3D` 上，用 `Depth + Cam` 反投影得到的 point map overall 为 `0.677`，优于 `MASt3R` 的 `0.826` 和 `DUSt3R` 的 `1.005`。
- 在 `ScanNet-1500` 两视图匹配上，`AUC@20 = 73.4`，超过 `Roma` 的 `70.9`。
- 训练好的 VGGT 特征还能增强 `CoTracker` 等下游 dynamic point tracking，并改善 feed-forward novel view synthesis。

## Strengths

- 它把多个传统上拆开的 3D task 压进一个统一 backbone，叙事非常完整。
- 结果不只是精度高，而且速度极强，feed-forward 模式下就能压过很多 optimization-heavy 方法。
- 几何归纳偏置并不重，却能学到非常强的多视图几何能力，这对 foundation model 方向很重要。
- 不只给最终数值，还证明特征可以迁移到下游 tracking / novel view synthesis。
- “先用大模型直接预测，必要时再接 BA refinement” 这个 recipe 很可能会被大量复用。

## Limitations

- 主战场仍然是静态场景几何；dynamic scene understanding 不是它的直接主目标。
- 虽然 feed-forward 已经很强，但 BA refinement 仍能继续提升，说明 optimization 还没有完全退出历史舞台。
- 第一帧参考系、中心 principal point 等设定说明它仍有一些工程化约束。
- 训练很重，依赖大规模异构 3D-annotated data mixture，这种 recipe 的复制门槛不低。
- 它更像 geometry backbone，而不是直接回答物理交互、可执行控制或 action dynamics 的模型。

## Questions

- `VGGT` 会不会像 `DINO` 之于视觉那样，变成很多 3D task 的默认 backbone。
- geometry backbone 和 3D generation latent 未来会不会合流，而不是长期分家。
- 如果加入时间、物理交互和动作，`VGGT` 这种 scene geometry model 能否自然长成更 general 的 world representation。
- 多任务联合监督的收益，主要来自 shared representation，还是来自多种 3D quantity 之间的闭式几何关系。
- 当 feed-forward geometry 模型越来越强时，传统 BA / SfM 会退化成 refinement layer，还是继续保留某些不可替代的 regime。

## My Take

`VGGT` 给我的感觉，不像一篇单点 trick paper，而像一篇“3D vision 也要进入 foundation model 阶段”的宣言。它最有价值的地方，是把一个原本高度 pipeline 化的领域，重新压缩成“共享 backbone + 多任务几何输出 + optional refinement”的形态。这种 recipe 一旦成立，后面很多 3D 感知、重建、跟踪、甚至生成工作，都会开始把它当成基础设施来看，而不是当成一个单独 benchmark winner。

如果 `TRELLIS` 更像是在问“3D asset 的 latent 表示该怎么统一”，那 `VGGT` 更像在问“真实场景几何本身能不能被一个大模型直接统一表达”。这两个问题不一样，但我觉得它们很可能会在未来某个点上接起来。
