# pi_0

## Summary

`pi_0` 代表了 VLA 路线里的一个明显转向：不再把重点放在“把动作离散成 token”，而是把重点放在如何用预训练 VLM backbone 加上连续 action 生成机制，去覆盖更高频、更灵巧、更真实世界的机器人控制。

## Problem

- 通用机器人策略不仅要泛化，还要处理复杂、连续、灵巧的动作
- 纯离散 token action 路线在高频 dexterous control 上可能不够自然
- 需要一个既继承 web-scale 语义知识、又能表达复杂连续动作分布的架构

## Main Idea

- 在预训练 VLM backbone 之上增加一个专门负责机器人输入输出的 action expert
- 使用 conditional flow matching 来建模连续动作 chunk
- 在多种 dexterous robot 平台的混合数据上训练 generalist robot policy

## Method

- 论文使用 PaliGemma 作为 VLM backbone
- 用一个较小的 action expert 处理 state 和 action token 位点
- 通过 flow matching 建模连续动作分布，而不是纯离散 autoregressive action token
- 论文报告该设计支持最高约 50 Hz 的 dexterous action chunk 控制

## Experimental Setup

- 数据来自多种平台，包括 single-arm、dual-arm 和 mobile manipulators
- 评估覆盖 direct prompting、语言指令跟随、高层 VLM policy 指挥下的执行，以及 fine-tuning 获得新技能
- 任务示例包括 laundry folding、table cleaning、assembling boxes

## Results

- 论文重点展示了跨平台 generalist policy 与 dexterous task 能力
- 项目后续开放日志显示，团队认为 `pi_0` 的主要使用方式是针对自有平台做 fine-tuning
- 官方 openpi 博客给出的经验是：一些任务只需约 1 到 20 小时数据即可完成有效 fine-tuning
- 在 2026-03-26 的 Sergey Levine seminar 中，这条线被进一步扩展为 `pi_0.5` 与 `pi*0.6`，重点转向 reasoning supervision 和 RL post-training

## Strengths

- 在 VLA 里更系统地引入连续动作建模
- 明显更贴近 dexterous manipulation 和高频控制需求
- 多平台训练和 action expert 设计都很值得后续研究复用

## Limitations

- 相比 OpenVLA，社区复现和研究生态起步更晚
- 训练 recipe 对数据质量和机器人平台的依赖可能更强
- 虽然后来开放了 openpi，但整体门槛仍高于“直接 fine-tune 开源 baseline”

## Questions

- flow matching action generation 相对离散 token action 的核心收益，到底来自表达能力还是训练 recipe
- 对普通学术组来说，`pi_0` 的可复现性和可迁移性有多强
- 如果把 OpenVLA 的开放生态和 `pi_0` 的连续动作建模结合，会不会更有研究价值

## Initial Take

初读视角下，`pi_0` 更像是 VLA 的“第二阶段”论文。RT-2 解决了范式问题，OpenVLA 解决了开放和研究进入门槛问题，而 `pi_0` 则在逼近更真实的控制需求，特别是 dexterous 和高频动作这一层。
后续 seminar 里最有意思的一点是，这条线并没有停在 action expert 上，而是继续往“thinking + RL post-training”的方向长，这说明 `pi_0` 更像起点而不是终点。
